BLACK PILLAR STATE — Operational Console

1. 3D Tele‑Map

This module shows the spatial layout of your State nodes. Each node has a position in ENU coordinates, a CID, a peer ID, and a signal mode.

How to use: Paste the MATLAB code into CoCalc or MATLAB and run it. You will see a 3D map with labeled nodes.

2. Tele‑URL Scheme

This module generates URLs that link each node’s CID, peer ID, and signal mode. These URLs act as “telemetry endpoints” for your State.

How to use: Inspect nodes(k).url_http and nodes(k).url_ipfs in MATLAB.

3. Radiograph View

This module slices the 3D map by altitude (U‑axis) to show vertical layers of the State. It is useful for diagnostics and structural clarity.

How to use: Run the radiograph code block in MATLAB to see each U‑layer.

4. Witness Log Signal

This module records events over time. It is the “heartbeat” of the State and shows continuity and activity.

How to use: Run the witness log code to generate a time‑series plot.

5. Unparallel Bridge

This module fuses physical, legal, signal, and temporal data into a single structure. It is the backbone of your jurisdictional model.

How to use: Run the bridge code and inspect the bridge structure.

6. Spherical Coordinate Envelope

This module creates a spherical “depth envelope” around your State node. It is used for detachment‑radius detection and oscillation‑event recognition.

How to use: Run the spherical envelope code in MATLAB to visualize the radius.

7. Vertical Pole Attachment

This module creates a vertical pole aligned with your robot’s spine. It represents your State‑location GPS signal and acts as a jurisdictional anchor.

How to use: Run the pole code to visualize the alignment.

8. Detachment Radius Monitor

This module detects when your presence‑signal moves outside the allowed radius. It logs the event and updates the witness log.

How to use: Run the radius‑check function with your current coordinates.

9. Oscillation Event Recorder

This module detects oscillatory movement patterns and records them as jurisdictional events.

How to use: Feed time‑series position data into the oscillation detector.
